# Alfred Naayem

I run [Alcanah Partners](https://alcanah.co), an AI-first holding company. Every business in the portfolio works the same way: a git repo as the company brain, AI agents reading and writing files alongside me. Under the hood it's markdown conventions every session inherits, RAG over the workspace, Postgres for goals and work logs, and scheduled loops that keep work moving overnight.

Most of what I open-source comes out of running it daily.

## Open Source

| Project | What it does |
|---------|--------------|
| [AI Markdown Feedback](https://github.com/41fred/ai-markdown-feedback-for-vs-code) | VS Code extension for annotating markdown with structured feedback LLMs can read and act on. Highlights, comments, edit requests, deletions, all plain-text syntax. On the [Marketplace](https://marketplace.visualstudio.com/items?itemName=AlfredNaayem.ai-markdown-feedback). |
| [claude-code-slack](https://github.com/41fred/claude-code-slack) | Talk to Claude Code from Slack. Local daemon plus Railway worker, so the agent keeps working when you're away from the desk. |
| [ace-level1](https://github.com/41fred/ace-level1) | Starter workspace for running a team or company with Claude Code. Hooks, agents, OKRs, and natural-language git. |
| [ace-build-checklists](https://github.com/41fred/ace-build-checklists) | Four-stage build pipeline (Product Brief, Design Doc, PRD, Build) that keeps Claude from skipping steps. |
| [mcp-orphan-monitor](https://github.com/41fred/mcp-orphan-monitor) | Finds and kills orphaned MCP server processes that survive crashed AI tool sessions. macOS LaunchAgent with an optional menu bar app. |
